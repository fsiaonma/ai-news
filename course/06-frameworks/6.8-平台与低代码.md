# 低代码平台

## 学习目标

- 了解主流低代码 LLM 平台的能力边界
- 判断 PoC 用平台、核心业务自研的分工
- 避免平台锁定与数据合规风险

## 引言

平台适合**验证想法**与**非核心流程**；深度定制、合规隔离、复杂 Agent Harness 仍需自研。

## 章节正文

### 第 1 步：平台对比

| 平台 | 强项 | 注意 |
| --- | --- | --- |
| **Dify** | 开源、Workflow + RAG + Agent | 需自托管运维 |
| **Coze / 豆包** | 快速 Bot、插件生态 | 云端、企业合规评估 |
| **RAGFlow** | 深度文档解析、版面分析 | 偏 RAG 场景 |
| **FastGPT** | 国内知识库问答 | 与业务系统集成深度 |
| **AutoGen Studio** | 多 Agent 可视化 | 研究向，生产需加固 |

**Ollama** 更偏本地模型运行时，不是应用平台，常与 Dify 组合。

### 第 2 步：何时用平台、何时自研

**适合平台**：
- 内部 FAQ、运营活动 Bot
- 2 周内要 demo
- 非敏感数据、可接受 SaaS

**必须自研**：
- 金融/医疗合规与审计
- 与 ERP/CRM 深度双向写
- 自定义 Harness、细粒度 RBAC
- 需要与 CI/CD、Eval 门禁一体化（第 7 模块）

常见路径：**平台 PoC → 验证 Prompt/RAG → 核心链路用代码重写**。

### 第 3 步：导出与迁移

选用平台前确认：

- Prompt、知识库文档能否批量导出
- Workflow 能否映射为 LangGraph 节点
- API 是否 OpenAI 兼容（降低迁移成本）

避免「只能 GUI 配置、无 API」的黑盒。

## 动手练习

1. 选一个平台注册试用，完成「上传 PDF + 对话」最小闭环
2. 列出迁移到自研栈时需导出的 5 类资产

## 本节小结

低代码平台加速 PoC；核心业务、合规与 Harness 自研。选型时看导出能力、API 兼容与数据驻留。

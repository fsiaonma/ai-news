# Trae 配置

## 学习目标

- 了解 **Trae**（字节跳动 AI IDE）的配置入口
- 设置项目 **Rules** 与 Builder / SOLO 模式
- 规划多 Agent 协作与国内网络环境下的 MCP

## 引言

**Trae** 是面向中文开发者的 AI IDE，提供 **Chat、Builder、SOLO** 等模式，内置模型与插件生态。配置思路与 Cursor 类似：**项目规则 + 上下文 + 工具**，但目录名与设置项以 Trae 当前版本为准（可在 Trae 内「设置 → 规则 / 智能体」查看）。

> Trae 迭代较快，本章给出配置 **框架**；具体菜单名称请对照 Trae 官方帮助文档。

## 章节正文

### 第 1 步：项目规则（Rules）

Trae 支持 **项目级 Rules**，通常可在：

- 项目根 **`.trae/`** 或设置界面「项目规则」中维护（以实际版本为准）
- 与 **README / 技术栈说明** 一并作为 Builder 上下文

建议规则内容：

```markdown
## 技术栈
VitePress + Node 20

## 必须
- 中文用户可见文案
- 改 course/ 源文件后运行 npm run sync

## 禁止
- 提交 API Key
- 直接改 website/ 下 sync 生成文件
```

若 Trae 支持 **文件 glob 规则**，可按 `course/**`、`scripts/**` 拆分，对齐 Cursor 的 `frontend.mdc` 做法。

### 第 2 步：Builder vs Chat vs SOLO

| 模式 | 适用 |
| --- | --- |
| **Chat** | 问答、小范围补全 |
| **Builder** | 从需求生成多文件、跑终端 |
| **SOLO / Agent** | 长任务自主规划（名称以产品为准） |

Multi-Agent 实践：

1. **Chat** 澄清需求与验收标准
2. **Builder** 生成初版代码
3. **SOLO** 跑通测试与修复（或人工 + Chat 迭代）

在团队文档中写清 **何时禁止 SOLO 自动提交**，与第七章生产规范一致。

### 第 3 步：Skills 与插件

Trae 的「技能 / 插件 / MCP」入口在设置或扩展市场：

- **内置插件**：Git、终端、预览
- **MCP**（若已支持）：配置方式类似 Cursor 的 `mcp.json`，连接内部服务时注意 VPN 与域名

团队可维护 **`docs/trae-playbook.md`**，记录：

- 推荐插件列表
- 常用 Builder prompt 模板
- 与 Cursor 规则差异说明

### 第 4 步：多 Agent 协作

Trae 多 Agent 可能体现为：

- **并行子任务**（探索 + 实现）
- **不同模型分工**（快模型写草稿、强模型审查）

配置策略：

1. 在 Rules 中定义 **探索阶段只读**
2. 实现阶段允许写文件与跑 `npm run build`
3. 审查阶段只读 diff，输出 checklist

与 **`AGENTS.md`** 角色表保持一致，便于团队在 Trae / Cursor 间切换。

### 第 5 步：国内环境注意点

- 模型与插件下载：预留网络与镜像策略
- 企业内网 MCP：优先自建 Server，避免公网依赖
- 数据合规：敏感代码禁用云端训练选项（按 Trae 隐私设置关闭）

### 第 6 步：与其他工具对齐

维护 **单一事实来源**：

```text
AGENTS.md          # 角色分工（全工具共用）
docs/agent-playbook.md
.cursor/rules/     # Cursor 细则
CLAUDE.md          # Claude Code / Codex 命令
README.md          # 人类 + Trae Builder 入口
```

Trae Rules 摘取 playbook 核心条目，避免四套完全独立文档。

## 小结

- Trae 锚点：**项目 Rules + Builder/SOLO 模式 + 插件/MCP**
- Multi-Agent 靠模式分工 + 与 AGENTS.md 对齐的角色定义
